Jsi expert na Three.js. Vytváříme jednoduchou single-file HTML demo (~50-100 KB) v pixel art / 8-bit retro stylu pro B2B pitch.

### Celkový styl
- Low-poly + pixelové textury (nebo Canvas texture s pixel art).
- Omezená barevná paleta (černá, zelená, oranžová, modrá, bílá, červená).
- Silné outline / scanline efekty pro retro vzhled.
- Orthogonal nebo lehce izometrická kamera (jako staré izo hry).

### Scéna
- Vlevo: CNC plotter (jednoduchý box + hlava, která se pohybuje).
- Vedle plotru: Postava operátora (muž s červenou kšiltovkou, ryšavý plnovous, černé šortky, hrnek kávy v ruce). Statická na začátku, pak jemně poskakuje a usmívá se.
- za plotrem: Grid 4x4 nebo 3x5 na "zdi" (jako fragmentovaný disk).
- V popředí vlevo: Stůl s PC (tower, monitor, klávesnice).

### Fáze animace (celkem ~25–35 sekund loop)

1. **Chaos fáze (8–10s)**  
   - Z monitoru PC vylétávají chaotické červené/oranžové částice (nebo malé pixelové bloky).  
   - Plotter "vyrábí" ploché obdélníkové panely (PET felt simulace – tenké boxy).  
   - Panely vylétávají náhodně a přistávají chaoticky na gridu (různé rotace, překrývání).  
   - Nad plotrem stoupají otazníky (?) a mizí.

2. **Transformace (1.5–2.5s)**  
   - Parser aktivace: [zde použij jednu z variant – ideálně energetická sféra nebo data beam z PC].  
   - Silný bílý/modře-bílý záblesk + časová dilatace (zpomalení pohybu).  
   - Všechny chaotické prvky se na chvíli zastaví.

3. **Uspořádaná fáze (10–12s)**  
   - Panely vylétávají z plotru uspořádaně, rotují se do správné pozice a elegantně zapadají do gridu jako Tetris/def ragmentace.  
   - Nad panely se objevují zelené symboly: ⏱, $, CSV, ✓.  
   - Datový tok z PC je nyní čistý zelený proud.  
   - Operátor se usmívá a lehce poskakuje.

### Technické požadavky
- Použij Three.js (import přes CDN pro single file).
- Animace pomocí requestAnimationFrame + GSAP nebo vlastní tweens.
- Optimalizace velikosti: co nejméně externích assetů, texture atlas, procedurální geometrie.
- Responsivní, mobile-friendly.
- Ovládání: tlačítka Replay, Pause, případně timeline scrubber.
- Komentáře v kódu pro snadnou iteraci.

Vygeneruj kompletní HTML soubor s komentáři. Nejdřív navrhni architekturu (scéna, objekty, animace timeline), pak kód.

**Analýza aktuální vývojové verze `systeq_threejs_demo.html`**

**Celkové hodnocení: 8.7/10**  
Demo je už **velmi solidní** a vizuálně působivé. Má silnou retro 8-bit / Norton Commander atmosféru, dobrý pacing a většinu klíčových prvků z původního zadání. Je to už použitelný pitch tool.

### Silné stránky

- **Atmosféra & Styl**: Výborný retro DOS/Norton Commander HUD, CRT efekty, EGA paleta, Press Start 2P font – perfektně sedí k tématu.
- **Technická kvalita**: Single HTML, rozumná velikost (~55 kB), dobré použití Three.js + CSS2D, optimalizace (low-poly, canvas texture na monitoru).
- **Choreografie**:
  - Chaos fáze (0–7s) funguje dobře – chaotické kostky, otazníky, vadné panely.
  - Aktivační flash + shockwave (7–10s) je dramatický.
  - Defragmentace + uspořádání panelů (10–22s) je viditelná a uspokojivá.
  - Operátorova emoce (smutný → veselý) je implementovaná.
- **HUD dynamika**: Živý cluster map, entropy, savings, progress – výborně podporuje příběh.
- **Interaktivita**: OrbitControls, F2 restart, klávesy.

### Slabiny / Problémy (prioritizované)

1. **Parser vizualizace je slabší**  
   Používá torus + shockwave sféru, ale není dost "energetický" a ikonický. Chybí jasný "Terminátor-like core" moment.

2. **Panel chování**  
   - V chaos fázi panely létají chaoticky, ale defragmentace v Order fázi není tak dramatická, jak by mohla být.
   - Některé panely zůstávají červené i po healu (barva se mění, ale efekt není vždy viditelný).

3. **Výkon & Čistota kódu**
   - V `animate()` je hodně logiky – při delším běhu se hromadí objekty (i když se mažou).
   - Některé funkce (`qmLbl`, `dcGeo`) jsou definovány uvnitř `triggerManualReset` – špatně čitelné.
   - `energeticFilamentsGroup` se tvoří nově každým framem v ACTIVATION → garbage.

4. **Malé detaily**
   - Operátorova animace radosti je trochu chaotická (skákání do stran).
   - Zlaté dolary a metric texty jsou hezké, ale mohou být častější.
   - Kamera je statická na začátku – chybí lehké automatické pohyby v klíčových momentech.
   - Žádný "Replay" button v UI (jen F2).

```js
// Nový Parser Core
let parserCore;
function createEnhancedPortal() {
  // ... stávající ...

  // Parser Core (CPU-like glowing cube/orb)
  const coreGeo = new THREE.IcosahedronGeometry(0.18, 1);
  const coreMat = new THREE.MeshBasicMaterial({
    color: EGA.WHITE,
    transparent: true,
    opacity: 0.9,
    blending: THREE.AdditiveBlending
  });
  parserCore = new THREE.Mesh(coreGeo, coreMat);
  parserCore.position.copy(portalGroup.position);
  scene.add(parserCore);
}
```

#Modulární disekce
**✅ SYSTEQ Three.js Demo – Modulární Dissekce (Grok Imagine / Iterative Dev Style)**

Toto je **nová analýza** inspirovaná metodikou vývojářů sdílejících postupy na X (modulární breakdown scén, low-poly foundation + overlays, declarative components, timeline-driven phases, object pooling, separace vizuálních systémů). Cílem je umožnit **nezávislou iteraci** na jednotlivých segmentech bez rozbíjení celku.

### 1. Celková Architektura Scény (High-Level Map)

**Hlavní vrstvy (nezávislé):**
- **HTML/CSS Overlay Layer** → Retro DOS/Norton HUD + CRT efekty
- **Three.js Core** → Scéna, kamera, renderer, lights
- **3D Objects / Groups** → Semanticky rozdělené (Wall, PC, Operator, Plotter, Portal)
- **Animation Systems** → Phase Manager + Spawnery + Update Loops
- **Data Flow / HUD Sync** → Metrics, Cluster Map, Monitor Canvas
- **Utility Layer** → Helpers, Materials, EGA Palette, Reset

**Klíčové konstanty (upravitelné na jednom místě):**
- `PHASE = { CHAOS: 0, ACTIVATION: 7, ORDER: 10, IDLE: 22 }`
- `EGA` paleta, pozice (`GND`, `WALL_Z`, atd.)

### 2. Modulární Segmenty Scény (pro iterativní vývoj)

#### **Segment A: Visual Foundation & Styling** (CSS + Renderer)
- `#crt` – CRT scanline + flicker
- `#flash-overlay` – Bílý magický flash (parser activation)
- `.dos-panel`, `.top-bar`, `.right-sidebar`, `.bottom-keys` – Norton Commander UI
- `.css-label` – Floating 3D texty (CSS2D)
- **Renderer setup**: WebGL (no AA pro pixel look) + CSS2DRenderer

**Iterační tip**: Změny v retro estetice dělej zde (barvy, scanlines, fonty).

#### **Segment B: Core Three.js Setup** (init())
- Scéna, kamera (45°), renderer
- Lights (Ambient + 2 Directional + Point)
- Controls (Orbit + damping)
- Floor + GridHelper
- Globální proměnné: `dataCubes`, `panels`, `questionMarks`, `goldenDollars`, `floatingTexts`

#### **Segment C: Statické 3D Objekty (createXXX funkce)**

1. **`createWall()`**  
   - Ornamentální frame + inner grid (3×4 slots)  
   - Cíl pro všechny panely

2. **`createPC()`**  
   - Desk + Tower + Monitor (CanvasTexture pro live DOS screen)  
   - Klávesnice, LEDky

3. **`createOperator()`**  
   - Group s head, body, arms, legs, cap, beard, eyes, smileMesh

4. **`createPlotter()`**  
   - Bed + rails + `plGantry` + `plSpindle` + laser light  
   - Hlavní pohybující se část (X/Z sliding)

5. **`createPortal()`** (Parser vizuál)  
   - Torusy, vortex rays, shockwave sphere  
   - `energeticFilamentsGroup` (propojovací linky)  
   - **Zde je hlavní prostor pro vylepšení parseru**

#### **Segment D: Dynamic Spawners** (Procedurální objekty)

- `spawnChaosDataCube()` → Chaotické barevné kostky z PC
- `spawnOrderedDataCube()` → Zelené čisté pakety (Order fáze)

- `spawnPanelOnPlotter(isDefective)` → Nejdůležitější: PET panely s `userData` (targetPos, chaosOffset, gridProgress, isDefective…)

#### **Segment E: Phase Manager & Animation Logic** (`animate()`)

Rozděleno na:

- **C. ORDER (10–22s)**: Defragmentace panelů (`easeOutBack`), heal barev, happy operator, green laser, ordered spawns
- **D. IDLE (>22s)**: Stabilizace + autoRotate

**Společné updaty** (mimo fáze):

#### **Segment F: HUD & Data Synchronization**

#### **Segment G: Reset & Utilities**
