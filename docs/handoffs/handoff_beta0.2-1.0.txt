## Instructions

- **Design language is LIGHT, not dark.** The design spec (`SYSTEQ_v1.0_BETA_Graficka_esteticka_vrstva.md`) defines: off-white background (#F9F9F9), orange/amber accent (#E67E22), Space Grotesk (UI) + JetBrains Mono (data/technical), 12/8/4 responsive grid, 8px vertical rhythm. This replaces the dark/teal visual identity of beta 0.2.
- **Parser demo = priority #1.** It must have 3 role-specific views (CEO = Kč/ROI, CTO = tables/JSON/API, Design = SVG toolpath canvas + layers) switchable via tab bar.
- **State machine**: IDLE → FILE_SELECTED → PARSING (loading) → RESULTS (with role tabs) → ERROR (with retry + fallback demo data).
- **No external dependencies.** Pure HTML5 + CSS3 + vanilla JS. Own inline SVG icons. Google Fonts (Space Grotesk + JetBrains Mono) via @import is allowed.
- **Multi-layer architecture**, not one-page: `/`, `/stack`, `/parser/vcf`, `/parser/dxf`, `/author`, `/solutions/{cnc,offgrid,iot}`, `/contact`, `/docs`. Each page = single purpose.
- **GitHub integration:** Create private repo `outpost2026/systeq-web` for version control, rollback, potential CI/CD. Never commit FTP passwords — use GitHub Secrets if automating.
- **Backend deploy is a separate chapter.** The front-end must work with fallback "demo data" JSON responses (hardcoded in JS) until real GCP Cloud Run endpoints are deployed.
- **High SNR** — every pixel has purpose. No carousels, no stock photos, no emoji as icons, no "under construction" badges.
